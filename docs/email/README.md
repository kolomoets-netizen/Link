# HTML-письма iStockLink

## product-update.html

Письмо для действующих пользователей о новых возможностях. Дизайн совпадает с типовым шаблоном iStock.Link (`tmpl-reference.html`).

### Тема письма

**Новые возможности iStock.Link**

(В теле письма заголовок не дублируется — сразу логотип и «Добрый день!»)

### Шрифт

Используется **системный стек** (как у Apple в интерфейсах):

`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Helvetica, Arial, sans-serif`

| Устройство | Что увидит получатель |
|------------|------------------------|
| iPhone / Mac | San Francisco |
| Windows | Segoe UI |
| Android | Roboto |
| Старый Outlook | Arial (запасной) |

Подключать Commissioner или Google Fonts через `@font-face` в письме **не стоит** — Gmail и Outlook часто их отбрасывают, письмо «прыгает» по виду. Системный стек — современно и без сюрпризов.

### Тема письма (варианты для A/B)

- **Новые возможности iStock.Link — формулы, тендеры, чаты**
- **Обновление iStock.Link: смотрите видео на Rutube**
- **Что нового в iStock.Link — последние релизы**

### Перед отправкой

1. Откройте `product-update.html` и замените плейсхолдеры:
   - `{{RUTUBE_VIDEO_ID}}` — ID ролика (32 символа из URL `https://rutube.ru/video/XXXXXXXX/`)
   - `{{layout_header_logo_url}}` — URL логотипа (как в системных письмах)
   - `{{layout_settings_unsubscribe_link}}` и `{{layout_settings_unsubscribe_text}}` — отписка

2. Отправьте **тестовое письмо себе** и проверьте в Gmail, Outlook и на телефоне.

### Как отправить через Gmail

Gmail не умеет «загрузить HTML-файл» напрямую. Два рабочих способа:

#### Способ 1 — копирование из браузера (проще всего)

1. Подставьте плейсхолдеры в `product-update.html` (ID Rutube, email поддержки).
2. Откройте файл в **Chrome** (перетащите файл в окно браузера или двойной клик).
3. Нажмите **Ctrl+A** (выделить всё) → **Ctrl+C**.
4. В Gmail нажмите **Написать**.
5. Кликните в поле текста письма → **Ctrl+V**.
6. Заполните **Кому**, **Тема** → отправьте **себе** для проверки.

Если вставилось «криво» — используйте способ 2.

#### Способ 2 — вставка HTML через DevTools (точнее)

1. В Gmail: **Написать** → в теле письма наберите слово `test` и выделите его.
2. Нажмите **F12** (инструменты разработчика) → вкладка **Elements**.
3. Найдите выделенный текст `test` внутри блока с `contenteditable="true"`.
4. Правый клик по родительскому `<div contenteditable="true">` → **Edit as HTML**.
5. Удалите содержимое и вставьте HTML **только из `<body>`** файла (без `<html>`, `<head>`, `<body>`).
6. Закройте DevTools, проверьте письмо, отправьте тест себе.

#### Несколько получателей

- До ~10–20 человек: добавьте адреса в **Кому** или **Скрытая копия (BCC)**.
- Массовая рассылка через Gmail не рекомендуется: лимит ~500 писем/сутки, риск попадания в спам. Для списка клиентов лучше Unisender / SendPulse.

### Другие способы отправки

- **Unisender / SendPulse / Mailchimp** — вставьте HTML в редактор «свой HTML» / «код письма».
- **Из приложения** — если есть SMTP/API, передайте HTML-тело как `text/html`.
- **Outlook (десктоп)** — «Вставить» → «Вставить как текст» обычно хуже; лучше «Вставить специальную → Форматированный текст (HTML)».

### Локальный просмотр

Откройте файл в браузере (двойной клик или `file://`). Для проверки с подставленным ID:

```bash
sed 's/{{RUTUBE_VIDEO_ID}}/ВАШ_ID/g; s/{{SUPPORT_EMAIL}}/support@example.com/g; s|{{UNSUBSCRIBE_URL}}|#|g' \
  docs/email/product-update.html > /tmp/preview.html && xdg-open /tmp/preview.html
```
